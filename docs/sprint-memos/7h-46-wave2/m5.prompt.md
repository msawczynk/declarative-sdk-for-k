Operates on `~/.cursor-daybook-sync/`. No SDK worktree.

Goal: author `~/.cursor-daybook-sync/scripts/codex_wave_launcher.sh` — a wrapper that fires N codex workers from a YAML manifest in parallel. This is one of the "extreme plan v3" automation scripts that makes Wave3+ truly low-touch.

# Hard requirements
1. NEW `~/.cursor-daybook-sync/scripts/codex_wave_launcher.sh`:
   - Usage: `codex_wave_launcher.sh <wave-manifest.yaml>`.
   - Manifest schema (yaml):
     ```yaml
     wave: "7h-46-w3"
     workers:
       - id: "L1a"
         prompt: "/tmp/dsk-w2-l1a.md"
         cwd: "/Users/martin/.../worktrees/cursor-l1a-pam-machine"
         mode: "offline" | "live" | "readonly"
       - id: "M2"
         prompt: "/tmp/dsk-w2-m2.md"
         cwd: "/Users/martin/.cursor-daybook-sync"
         mode: "offline"
     ```
   - For each worker: dispatch to `codex_offline_bg.sh` (mode=offline|readonly) or `codex_live.sh` via nohup (mode=live).
   - Live workers: `nohup bash codex_live.sh <prompt> > /tmp/codex-live-<wave>-<id>.log 2>&1 & disown`.
   - Output one line per worker: `LAUNCHED <id> mode=<mode> log=<path> marker=<path|n/a>`.
   - Exit 0 if all dispatched cleanly, non-zero on any dispatch failure.

2. Author `~/.cursor-daybook-sync/scripts/test_codex_wave_launcher.sh`:
   - Smoke test: feed a manifest with 2 mock workers (using `echo` as fake codex via `CODEX_BIN_OVERRIDE` if supported, else minimal stub).
   - Assert all dispatched + log files created.

3. Add usage docs at top of script: `# Usage: ...`.
