1. Add `_codex_worker_counts(boot_time: float, marker_dir: Path = Path("/tmp")) -> dict[str, int]` to `agent_review_loop.py`:
   - Glob `/tmp/codex-offline-*.marker` and `/tmp/codex-live-*.marker`.
   - Filter mtime >= boot_time.
   - Parse STATE= lines from each marker.
   - Return `{codex_workers_session: int, codex_workers_done: int, codex_workers_launched: int}`.
2. Wire into the metric output JSON: add `codex_workers_session`, `codex_workers_done`, `codex_workers_launched`.
3. Wire into the markdown digest: a row showing the codex counts.
4. KEEP `subagent_transcripts` separate (Cursor Task subagents only).
5. Update `test_pending_count.py` (or NEW `test_codex_worker_counts.py`) with R1's test cases:
   - Old marker (mtime < boot) does NOT count.
   - New DONE marker counts as session + done.
   - New LAUNCHED marker counts as session + launched.
   - New live marker counts as session.
   - Malformed marker counts as session, not done/launched.
6. Run tests: `python3 ~/.cursor-daybook-sync/scripts/test_codex_worker_counts.py` (or `test_pending_count.py`).
7. Sanity-check: run `bash ~/.cursor-daybook-sync/scripts/agent_review_loop.sh` once and confirm new metrics appear in `review-latest.json`.
